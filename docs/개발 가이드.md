## 설계 원칙

### 백엔드 추상화 (Repository 패턴)
- 모든 데이터 접근은 `repositories/interfaces.ts`에 정의된 인터페이스를 통해 수행
- 현재 구현체: `repositories/firebase/` (Firestore)
- 백엔드 변경 시: 새 구현체 작성 후 `repositories/index.ts`에서 교체

### 디자인 방향
- 모바일 우선 (주 사용 기기: 스마트폰)
- 태블릿 대응 (max-width: 768px)
- 컬러 톤: 요넥스 스타일 (블루 #0066B3 + 그린 #00A651)
- 주요 액션 버튼: 하단 플로팅 고정
- UI 인터랙션 규칙: `docs/UI 인터랙션 가이드.md` 참고

### 페이지 구조
- `(main)` 그룹: 공통 헤더(EveryMinton 로고) + 네비게이션(일정/모임원 탭)
- `schedule/[id]`: 자체 헤더(❮ 백버튼 + 날짜) + 탭(코트/대기/참여자/매칭설정/일정정보) + 조회 링크 복사 버튼
- `view/schedule/[id]`: 공유용 조회 전용 화면(운영/관리 기능 없음)

- Node.js v18 이상 (현재: v25.2.1)

```bash
cd app

# http://localhost:3000 에서 확인
```

```bash
cd app

```
