# UI 인터랙션 가이드

앞으로 모든 바텀시트, 팝업, 토스트 구현 시 아래 규칙을 따른다.

---

## 바텀시트 (Bottom Sheet)

### 닫기 방식
- **X 버튼**: 타이틀 우측에 ✕ 버튼 배치 (하단 "취소" 버튼 사용하지 않음)
- **배경 탭**: 바텀시트 외부(오버레이) 탭 시 닫힘
- **아래로 드래그**: 바텀시트 전체 영역에서 아래로 드래그 시 닫힘 (100px 이상 드래그 시 닫힘, 미만이면 스냅백)
- **안드로이드 백버튼**: 바텀시트가 열릴 때 `history.pushState` → 백버튼 시 바텀시트만 닫힘 (페이지 이동 방지)

### 드래그 구현 규칙
- state가 아닌 DOM 직접 조작 (useRef) 방식으로 구현하여 끊김 방지
- 스크롤이 맨 위일 때만 드래그 닫기 활성화 (스크롤과 충돌 방지)
- 위로 드래그 시 드래그 모드 해제하여 일반 스크롤 허용

### 구조
```
[오버레이 (bg-black/50)]
  └─ [바텀시트 컨테이너 (rounded-t-2xl)]
       ├─ 핸들 바 (w-9 h-1 회색 둥근 바)
       ├─ 헤더 (타이틀 + ✕ 버튼)
       ├─ 콘텐츠
       └─ 액션 버튼
```

## 토스트 (Toast)

### 표시 위치
- 화면 하단 (bottom-10)

### 애니메이션
- 나타날 때: 아래에서 위로 슬라이드 업
- 사라질 때: fade out (0.5초)

### 타이밍
- 2초 표시 → 0.5초 fade out → 제거

### 스타일
- 반투명 80% 배경
- backdrop-blur 사용하지 않음 (성능 이슈)
- 타입별 색상: error(빨강), success(그린), info(블루)

### 사용
- 시스템 기본 `alert()` 사용 금지
- `useToast()` 훅으로 호출: `showToast("메시지", "error")`

## 드롭다운 메뉴

### 위치
- 트리거 버튼 기준 바로 아래 또는 위에 표시 (top-12 / bottom-12)
- 화면 하단 공간이 120px 미만이면 위로 열림 (자동 판단)

### 닫기
- 메뉴 외부 탭 시 닫힘 (fixed inset-0 투명 오버레이)

### 주의
- 부모 컨테이너에 `overflow-hidden` 사용 금지 (드롭다운이 잘림)

---

## 공통 원칙

- 시스템 기본 UI (`alert`, `confirm`) 사용 금지 → 커스텀 컴포넌트 사용
- 모든 모달/바텀시트는 안드로이드 백버튼 대응 필수
- 성능에 영향을 주는 CSS 효과 (backdrop-blur 등) 지양
